このファイルは、Codex / Claude / その他AIエージェントが、既存サイトのデザインを修正・拡張するときに参照するための基準です。

それらは `AGENTS.md`、`NOTICE`、または別の運用ルール文書で管理します。

## 1. デザインコンセプト

このサイトは、ECサイトではなく、静かな展示室・個人アーカイブ・ブランドサイトのように見せる。

目指す体験は、商品を素早く探すことではなく、世界の気配を眺めながら静かに辿ること。

デザイン上の中心思想は以下。

> もう戻れない世界の気配を、日常のそばに残す。

画面は、売り場ではなく、収蔵庫。  
カードは、商品棚ではなく、展示ラベル。  
リンクは、購入ボタンではなく、外部への静かな出口。  
背景は、壁紙ではなく、空気。  
スクロールは、移動ではなく、記録をたどる進行。  
ホバーは、色の変化ではなく、触れたときの小さな発見。

このサイトでは、情報をただ配置しない。

情報に触れたとき、画面がどう反応するかまで設計する。

最重要原則は以下。

> 情報を配置するのではなく、情報に触れたときの反応まで設計する。

---

## 2. デザインの基本思想

### 2.1 静けさを優先する

このサイトは、騒がしいサイトではない。

派手な装飾、強い色、過剰なアニメーション、強い購入導線、情報過多のレイアウトを避ける。

デザイン全体は、静かで、余白があり、眺めていたくなる佇まいを目指す。

### 2.2 店ではなくアーカイブとして見せる

このサイトは、商品を売り込む場所ではない。

商品は、売るものではなく、収蔵されたオブジェクトとして扱う。

価格、在庫、セール、ランキング、おすすめ度を主役にしない。

### 2.3 CVRより空気感を優先する

一般的な商用サイトでは、CVR、購入導線、商品到達速度、離脱防止、ランキング、比較表が重視される。

しかし、このサイトではそれらを最優先にしない。

優先するのは、世界観、美意識、余白、静けさ、余韻、個人編集感、眺めていたくなる体験である。

### 2.4 余白を恐れない

余白は空白ではない。

このサイトにおける余白は、廃墟の静けさ、展示室の距離感、世界観の余韻を作るための重要な要素である。

情報を詰め込みすぎず、オブジェクトが静かに置かれている感覚を作る。

### 2.5 説明しすぎない

すべてを一度に説明しない。

触れたときに少しだけ分かる。  
スクロールしたときに少しだけ現れる。  
選んだときに世界の断片が近づく。

そのような発見の余地を残す。

### 2.6 テンプレート感を避ける

このサイトは、一般的なブログテンプレートやアフィリエイトテンプレートの見た目に寄せない。

人の感性で編集された、静かな個人アーカイブのように見せる。

## 7. タイポグラフィ方針

### 7.1 全体方針

文字は、静かで、読みやすく、編集された印象を持たせる。

大きな英字見出しは使ってよいが、派手にしない。

日本語本文は読みやすさを優先する。

方針は以下。

- 大きな英字見出し
- 小さめだが読める日本語本文
- 広めの行間
- 文字数は少なく
- 読ませるより、余韻を置く
- 装飾的な英字ラベルは小さくしてよいが、判読不能にはしない

### 7.2 推奨フォント

本文・UIの基本フォントは以下を想定する。

    font-family:
      "Inter",
      "Noto Sans JP",
      "Hiragino Sans",
      "Yu Gothic",
      system-ui,
      sans-serif;

大きな英字見出しで、より編集感を出したい場合は以下も検討する。

    font-family:
      "Cormorant Garamond",
      "EB Garamond",
      "Times New Roman",
      serif;

### 7.3 英字見出し

英字見出しは、静かな古さ・記録感・展示感を持たせる。

推奨方向は以下。

- serif
- high contrast
- editorial
- large scale
- generous spacing

例。

    SEED A HOSTILE EARTH
    WORLDS
    OBJECTS
    JOURNAL
    SELECTED RELICS
    QUIET ARCHIVE
    FRAGMENTS
    AFTERGLOW

### 7.4 日本語本文

日本語本文は読みやすさを維持する。

- 行間は広め
- 1段落は短く
- 説明しすぎない
- コピーは1〜3行で止める
- 長文にしない
- 商品説明文は説明書きではなく、展示文に近づける

## 8. 最低可読性ライン

このサイトはデザイン優先の個人アーカイブであるが、文字が読めない・小さすぎる・操作しづらい状態は許容しない。

静けさ、余白、展示室感を優先しつつ、最低限の可読性を必ず確保する。

### 8.1 基本文字サイズ

本文の基準サイズは、PC・モバイルともに **16px以上** を原則とする。

CSS目安。

    body {
      font-size: 16px;
      line-height: 1.85;
    }

禁止。

- `body` を `15px` 未満にしない
- 日本語本文を `14px` 以下にしない
- 雰囲気を理由に本文全体を小さくしない
- 小さくて読めない状態を、静けさとして扱わない

### 8.2 本文・リード文のサイズ

通常本文、説明文、リード文は、以下を最低ラインとする。

| 要素 | 最低サイズ | 推奨サイズ | 備考 |
|---|---:|---:|---|
| 通常本文 | 16px | 16px〜17px | 日本語の可読性を優先 |
| リード文 | 16px | 16px〜18px | Hero下やConceptに使用 |
| 商品説明文 | 15.5px | 16px | 小さすぎないようにする |
| Journal本文 | 16px | 16px〜18px | 長文は特に読みやすさ優先 |
| ABOUT本文 | 16px | 16px〜18px | 思想文は読みやすくする |

CSS目安。

    .lead,
    .concept-text,
    .statement p,
    .journal-body,
    .about-body {
      font-size: clamp(1rem, 1.25vw, 1.08rem);
      line-height: 1.9;
    }

    .object-description,
    .world-card p,
    .type-card p {
      font-size: 1rem;
      line-height: 1.85;
    }

### 8.3 見出しサイズ

見出しは大きく、余白を持って見せる。

| 要素 | 最低サイズ | 推奨サイズ |
|---|---:|---:|
| Hero h1 | 48px | 56px〜88px |
| Page h1 | 40px | 48px〜72px |
| Section h2 | 28px | 32px〜48px |
| Card h3 | 18px | 19px〜22px |

CSS目安。

    .hero h1 {
      font-size: clamp(3rem, 7vw, 5.5rem);
      line-height: 0.95;
    }

    .page-hero h1 {
      font-size: clamp(2.5rem, 5vw, 4.5rem);
      line-height: 1;
    }

    .section-heading h2,
    .section h2 {
      font-size: clamp(1.9rem, 3.8vw, 3.2rem);
      line-height: 1.15;
    }

    h3,
    .world-card h3,
    .object-card h3,
    .type-card h3 {
      font-size: clamp(1.15rem, 1.7vw, 1.35rem);
      line-height: 1.35;
    }

### 8.4 小さな英字・メタ情報

英字ラベル、eyebrow、カテゴリ名、日付、番号などは小さくしてよい。  
ただし、読めないほど小さくしてはいけない。

| 要素 | 最低サイズ | 推奨サイズ |
|---|---:|---:|
| eyebrow | 11.5px | 12px〜13px |
| object number | 11.5px | 12px〜13px |
| meta text | 11.5px | 12px〜13px |
| checked date | 11.5px | 12px |
| breadcrumb | 12px | 12px〜13px |

CSS目安。

    .eyebrow,
    .object-number,
    .object-world,
    .object-checked,
    .breadcrumb,
    .last-updated,
    .meta,
    .card-meta {
      font-size: 0.75rem;
      line-height: 1.5;
      letter-spacing: 0.12em;
    }

禁止。

- `0.7rem` 未満を多用しない
- 英字ラベルを装飾として読めないサイズにしない
- 日付やカテゴリ情報を判読困難にしない

    2. Concept

### 10.2 リンク

リンクは、触れる対象として扱う。

通常状態では控えめに見せる。  
ホバー時には、触れたことが分かるように、静かな変化を返す。

推奨表現は以下。

- 細い下線
- 薄い背景
- 文字色のわずかな変化
- 下線が静かに伸びる
- 境界線がわずかに現れる
- 触れた場所に小さな反応が返る

避ける表現は以下。

- デフォルトの明るい青リンク
- 派手な色変化
- 点滅
- 発光
- 強い拡大
- うるさいホバー演出

### 10.3 外部リンク

外部販売先へのリンクは「出口」であり、主役ではない。

方針は以下。

- ボタンを大きくしない
- 強い背景色を使わない
- Amazon / Rakuten / Yahoo Shopping は控えめに表示
- 「今すぐ購入」は使わない
- 外部リンクはCTAではなく、静かな出口として扱う

サイズ目安。

| 要素 | 最低サイズ | 推奨サイズ |
|---|---:|---:|
| 外部リンク | 12.5px | 13px〜14px |
| filter button | 13px | 13px〜14px |
| text link | 13px | 13px〜15px |

CSS目安。

    .object-links a,
    .button-link,
    .filter-button,
    .text-link {
      font-size: 0.82rem;
      line-height: 1.4;
    }

### 10.4 ボタン

## 11. カード表現

カードはECカードではなく、展示ラベルに近づける。

### 11.1 基本方針

- 薄い罫線
- 控えめなhover
- 大きすぎないCTA
- タグは小さく薄く
- 商品よりも世界観・分類・短い説明を優先
- 価格や在庫を前面に出さない
- 広告カードのように見せない

### 11.2 Worldカード

Worldカードは、作品世界への扉として扱う。

通常のカテゴリカードではなく、異なる展示室への入口として見せる。

方針は以下。

- 商品カードより先に表示する
- 商品カードより少し存在感を持たせる
- lead文は読める大きさにする
- hoverは控えめにする
- 黒背景や派手な演出にしない
- ホバー時には、カテゴリをクリックする感覚ではなく、別の部屋に入る感覚を作る

推奨ホバー表現。

- 背景がわずかに変化する
- 境界線が静かに浮かぶ
- タイトルが少し鮮明になる
- 短い空気感のあるコピーが現れる
- テクスチャや画像がゆっくり見える

### 11.3 Objectカード

Objectカードは、収蔵品のラベルとして扱う。

商品カードではなく、記録、遺物、断片として見せる。

通常状態の方針は以下。

- 白または薄い面
- 細い境界線
- 広めの内側余白

## 12. インタラクション設計

### 12.1 基本思想

このサイトでは、Web上の情報を「触れる情報」として設計する。

リンク、カード、ボタン、見出し、背景、スクロール、ページ遷移などに対して、ユーザー操作への微細な反応を持たせる。

重要なのは、派手なアニメーションではない。

ユーザーの操作に対して、画面が意味のある返答を返すことである。

### 12.2 インタラクションで伝えるもの

動きは以下を伝えるために使う。

- 重要度
- 発見
- 選択
- 遷移
- 奥行き
- 余韻
- 世界観への接触感

### 12.3 インタラクションで避けるもの

以下は避ける。

- 装飾だけのアニメーション
- 意味のない回転
- 過剰な拡大
- 派手なバウンド
- 強い発光
- 点滅
- ゲーミング風の演出
- CTAを強調するためだけの動き
- 購入誘導を強める動き
- 操作速度を大きく落とす演出

## 31. デザイン要約

このサイトは、FromSoftware関連商品を売るためのサイトではない。

もう戻れない世界の断片を、静かに集め、余白の中に置き、必要な人が眺められるようにする個人編集アーカイブである。

ユーザーに「買わなければ」と思わせるのではなく、「少し見ていたい」「この気配を日常のそばに置きたい」と感じさせる。

最終的に目指す体験は以下である。

ユーザーが小さな展示室に入り、そこに置かれたオブジェクトに近づき、触れたとき、世界が静かに返事をする。

最終ルールは以下。

> 情報を配置するのではなく、情報に触れたときの反応まで設計する。
