# AGENTS.md

## 1. 目的

このファイルは、CodexなどのAIコーディングエージェント向けの作業ルールを定義する。

このプロジェクトは、FromSoftware作品に着想を得た、静的・デザイン優先・非商用感重視の個人アーカイブサイトである。

目的は、コンバージョン、クリック、アフィリエイト収益、商品露出を最大化することではない。

目的は、静かな視覚世界、編集的な文体、個人アーカイブとしての体験を保つことである。

---

## 2. 作業前に必ず読むファイル

変更を行う前に、以下のファイルを読むこと。

1. `DESIGN.md`
2. `data/site.json`
3. `data/products.json`
4. `scripts/build.js`
5. `style.css`

```text
ユーザーの明示指示
↓
DESIGN.md
↓
AGENTS.md
↓
既存実装
```

## 4. デザイン思想

このサイトは通常のアフィリエイトサイトではない。

ECサイトのように最適化しない。

### 優先しないこと

- CVR
- CTAの強化
- 購入までの速さ
- 商品ファーストのレイアウト
- 商品ランキング
- セールの見せ方
- 最安値比較
- アフィリエイトリンクの目立たせ
- 過剰なSEOテキスト
- 雰囲気を犠牲にした利便性

### 優先すること

- 視覚的な空気感
- 余白
- 静けさ
- 編集的なリズム
- 世界を先に見せる導線
- 個人アーカイブ感
- 戻りたくなる場所の感覚
- FromSoftware的な緊張感と残響
- ただし公式テキストの模倣や引用はしない

## 5. TOPの必須構成

ホームページは必ず以下の順序にする。

```text
1. Hero
2. Concept
3. Worlds
4. Object Types
5. Journal
6. Selected Relics
```

Hero直後に商品カードを配置しない。

商品カードは後半の `Selected Relics` セクションにのみ、少数だけ配置してよい。

## 8. 文体ルール

静かで、抑制された、編集的な文体を使う。

私的なアーカイブに断片を収蔵するように書く。

### 良い方向

- 短い
- 静か
- 断片的
- 空気感がある
- 抑制されている
- 少し儀式的
- 説明しすぎない
- 売り込みではない

### 避ける方向

- FromSoftware公式テキストの模倣
- 作中アイテム説明文のコピー
- 公式文の直接引用
- 過剰な中二表現
- ネットスラング
- 強い売り文句
- 商品レビュー文
- 一般的なアフィリエイトコピー

### 絶対に使わない表現

- おすすめ
- 人気
- ランキング
- セール
- 最安値
- 今すぐ購入
- 絶対買うべき
- 神商品
- 最強

---

## 9. 商品の扱い

商品は主役ではない。

商品は以下のように扱う。

| 一般的な扱い | このプロジェクトでの扱い |
|---|---|
| product / 商品 | object / オブジェクト |
| goods / グッズ | fragment / 断片 |
| figure / フィギュア | relic / 遺物 |
| art book / アートブック | record / 記録 |
| soundtrack / サウンドトラック | echo / 残響 |
| apparel / アパレル | mark / 記章 |
| game / ゲーム | entrance / 入口 |
| category / カテゴリ | form / かたち |
| external link / 外部リンク | quiet exit / 静かな出口 |

商品説明は1〜2文の短文にする。

商品種別が分かる範囲で、空気感・記憶・世界の断片としての性質を説明する。

## 11. 法務表記

法務・権利・非公式表記・アフィリエイト表記を削除しない。

法務文言は視覚的に控えめにしてよいが、意味を失ってはいけない。

ABOUT / NOTICE / footer など適切な場所で、以下を明確にすること。

- このサイトは非公式サイトであること
- FromSoftwareおよび各権利者とは関係がないこと
- 商品名、作品名、商標、画像等の権利は各権利者に帰属すること
- 商品画像はASP・ショップ・公式に許可された範囲でのみ使用すること
- 価格、在庫、販売条件は外部販売先で確認すること
- アフィリエイトプログラムを利用する場合があること

## 12. ファイル編集とビルドルール

生成済みHTMLだけを直接編集しない。

可能な限り、元データや生成元を編集する。

### 優先して編集するファイル

- `data/site.json`
- `data/products.json`
- `scripts/build.js`
- `style.css`

### 避けること

- 生成済みHTMLだけを修正する
- 複数の生成ページに同じ内容をハードコードする
- 商品データをHTML側に重複して持たせる

生成済みHTMLを修正する場合は、再ビルドで消えないよう、生成元も必ず更新する。

最低限の使いやすさは維持する。

## 16. 変更プロトコル

大きな構造変更を行う前に、まず計画を出す。

計画には以下を含める。

1. 現在の問題
2. 提案する変更
3. 編集対象ファイル
4. 期待される影響
5. リスク
6. 確認方法

ユーザーの承認なしに、大きな構造変更を実装しない。

小さなコピー修正やスタイル修正は、ユーザーが明示的に依頼した場合に直接実装してよい。

---

## 17. 禁止事項

ユーザーが明示的に依頼しない限り、以下を行わない。

- サイト名を変更する
- Heroコピーを変更する
- 商品カードをWorldsより上へ移動する
- ランキングを追加する
- セールバッジを追加する
- 強い購入CTAを追加する
- アフィリエイトリンクを目立たせる
- 通常のEC・アフィリエイトレイアウトへ変更する
- 法務表記を削除する
- SEKIROやDemon’s Soulsを削除する
- 公式サイトと誤認される表現を追加する
- この `AGENTS.md` を自動編集する
- ユーザーの明示指示なしに `DESIGN.md` を編集する

## 18. 確認チェックリスト

変更後、以下を確認する。

- サイト名が `SEED A HOSTILE EARTH` のままである
- TOP順序が Hero → Concept → Worlds → Object Types → Journal → Selected Relics のままである
- 商品がWorldsより前に表示されていない
- 6つのWorldsが存在する
- 商品カードが控えめに見える
- 禁止された販売文句が追加されていない
- 法務表記が残っている
- モバイル表示が崩れていない
- 生成HTMLが元データ・buildロジックと整合している
- `AGENTS.md` と `DESIGN.md` が、明示指示なしに変更されていない

## 20. 法務・権利表記の保護ルール

このプロジェクトでは、FromSoftware作品名、商品名、画像、商標、外部販売リンク、アフィリエイトリンクを扱うため、法務・権利表記をデザインや導線改善の都合で削除・弱体化してはいけない。

### 必ず維持する表記

以下の内容は、ABOUT、NOTICE、footer、または適切な法務表記エリアで確認できる状態を維持する。

- このサイトは非公式サイトであること
- FromSoftwareおよび各権利者とは関係がないこと
- 掲載している作品名、商品名、商標、画像等の権利は各権利者に帰属すること
- 商品画像は、ASP、ショップ、公式、権利者などが許可している範囲でのみ使用すること
- 外部リンクにアフィリエイトリンクを含む場合があること
- 価格、在庫、販売条件、商品仕様は外部販売先で確認すること

### 変更してはいけないこと

以下は、ユーザーが明示的に依頼しない限り行わない。

- 法務表記の削除
- 非公式表記の削除
- アフィリエイト表記の削除
- 権利帰属表記の削除
- footer / NOTICE / ABOUT から法務導線を消すこと
- 商品画像の権利確認を省略すること
- 公式サイトと誤認される表現を追加すること
- FromSoftware公式、または各作品公式と提携・承認・監修関係があるように見せること
- 公式ロゴ、公式コピー、作中テキスト、アイテム説明文を無断で模倣・転載すること

### デザイン上の扱い

法務表記は、サイトの静かな雰囲気を壊さない範囲で控えめに見せてよい。

ただし、控えめにすることと、読めない・見つけられない・意味が分からない状態にすることは別である。

法務表記は以下を満たすこと。

- 読める文字サイズである
- footer または NOTICE への導線がある
- ABOUT からも確認できる
- 外部リンクや商品紹介がある場合、アフィリエイトリンクを含む可能性を確認できる
- 商品価格、在庫、販売条件は外部サイトで確認する旨が分かる

### 判断基準

迷った場合は、以下を優先する。

1. 公式サイトと誤認されないこと
2. 権利者との関係を誤認させないこと
3. 商品画像・作品名・商標の権利帰属が確認できること
4. アフィリエイトリンクの可能性が確認できること
5. デザイン上は静かでも、表記の意味が失われていないこと
